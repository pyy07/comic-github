# 《漫说GitHub》公众号写作协作规范（v1.0）

## 1. 项目目标与定位
- 项目名：`漫说GitHub`
- 内容形态：面向公众号的系列文章，主打“GitHub 值得介绍仓库精选 + 通俗场景讲解”。
- 核心价值：帮非深度技术读者快速发现“有趣、能带来情绪价值、同时有现实用途”的开源仓库。
- 目标读者（暂定）：
  - 非深度技术人员（运营、产品、内容创作者、独立个体）
  - 轻技术背景的开发者、在校生、转码人群
- 风格关键词：`通俗`、`场景先行`、`少术语`、`直接可用`。

## 2. 内容边界（写什么 / 不写什么）
- 写什么：
  - `仓库精选`：围绕真实用户场景挑选 GitHub 仓库
  - `上手指引`：安装/注册/使用路径、最低门槛用法
  - `价值判断`：适用人群、使用成本、替代方案、风险提醒
  - `轻实操`：可直接复制的最小操作步骤
- 不写什么：
  - 只讲技术原理、没有用户场景的“炫技文”
  - 缺少试用验证的仓库推荐
  - 过度依赖复杂开发环境的重度教程（除非标注进阶）

## 3. 选题路线（12 周起步）
- 路线原则：
  - `80%` 场景驱动仓库精选（可长期复用）
  - `20%` 热点响应（新仓库、新版本、新趋势）
- 选题优先级（硬规则）：
  - `第一优先`：有趣 + 情绪价值（看完想分享、想收藏、想尝试）
  - `第二优先`：对非专业用户有明确场景价值
  - `第三优先`：Trending / Stars（仅作参考，不作为前置门槛）
- 仓库筛选标准（发布前必查）：
  - 先判断“是否有趣、能否提供情绪价值”
  - 有明确使用场景，且非深度技术人员可感知价值
  - Trending / Stars 只用于辅助背书，不作为必选条件
  - `README` 清晰、近期有维护、基础文档可读
  - 有可执行的最小上手路径（5-15 分钟内完成）
  - 明确标注成本与门槛（是否需要 API Key、是否付费、是否本地部署）
  - 至少满足其一：`马上有用`（可直接解决一个小问题）或 `强好奇心`（读者会想立刻点开体验）
  - 示例（情绪价值导向）：https://github.com/moeru-ai/airi
- 选题池（首批）：
  1. 不会写代码也能搭知识库：适合个人整理的 GitHub 开源工具
  2. 内容创作者提效：3 个可用来做图文/视频辅助的仓库
  3. 小团队项目管理：比表格更清晰的开源工具仓库推荐
  4. 个人效率自动化：非工程师也能用的任务自动化仓库
  5. 做副业落地页：可快速搭建页面的开源仓库精选
  6. 文件整理与搜索：本地资料管理类仓库实测
  7. AI 工具聚合：哪些仓库适合“轻使用”而不是“深开发”
  8. Markdown 工作流：写作人群最实用的开源仓库
  9. 简历/作品集制作：低门槛开源方案对比
  10. 数据可视化入门：非技术岗位可直接使用的仓库
  11. 远程协作工具箱：开源替代方案盘点
  12. 每月精选：本月 5 个最值得收藏的 GitHub 仓库

## 4. 单篇标准模板（建议固定）
- 标题：`场景 + 人群 + 结果`（示例：`运营同学也能用：3 个 GitHub 仓库帮你一小时做完周报素材`）
- 开场（100-150 字）：
  - 一个用户场景切入（谁、在什么情况下、遇到什么麻烦）
  - 明确“这篇会给你哪几种可选仓库”
- 主体结构：
  1. 用户场景拆解（需求、限制、预算、时间）
  2. 仓库速览卡（用途、门槛、优缺点、适用人群）
  3. 选型建议（什么人选 A，什么人选 B）
  4. 避坑提示（至少 3 条）
  5. 一页总结（Checklist）
- 结尾 CTA：
  - 引导评论区反馈“你想解决的具体场景”
  - 预告下一篇（形成追更）
- 配图规范（暂定）：
  - 每篇 3-6 张图，优先“场景图 + 界面对比 + 关键步骤截图”
  - 图注统一格式：`图N：动作 + 结果`

## 5. 发布节奏与生产流水线
- 频率（暂定）：每周 `2` 篇（周二/周五）
- 周期流程（单篇）：
  1. 周选题（D1）：确定标题、读者收益、关键词
  2. 打大纲（D1）：按模板产出一级二级结构
  3. 写初稿（D2）：先完整再打磨
  4. 技术校验（D2-D3）：命令可执行、截图可复现
  5. 编辑润色（D3）：压缩废话、统一术语
  6. 发布与复盘（D4）：收集数据并记录改进点
- 质量门槛（发布前必须满足）：
  - 固定推荐 `3` 个仓库，并给出选型理由
  - 以场景价值为核心，不强制实操教程
  - 每个仓库至少说明 1 个“马上能用”的具体场景
  - 至少 3 个常见坑与规避建议
  - 标题、导语、结尾 CTA 都已填写

## 6. 协作规则（人机协同）
- 用户负责：
  - 最终选题拍板
  - 个人经验与观点输入
  - 发布渠道与排期决策
- Agent 负责：
  - 仓库候选搜集与首轮筛选
  - 场景化结构设计、初稿产出
  - 上手步骤与风险提示校验清单
  - 标题备选、摘要、封面文案建议
- 协作约定：
  - 所有文章先产出 `Outline` 再写正文
  - 默认 2 轮迭代：`结构轮`、`文案轮`
  - 技术事实优先，表达风格次之
  - 不确定信息统一标记 `待确认`

## 7. 文件与目录约定（从现在开始）
- 建议目录：
  - `topics/`：选题池与排期
  - `drafts/`：文章草稿（按日期+slug 命名）
  - `assets/`：配图、流程图源文件
  - `templates/`：固定模板（大纲、正文、复盘）
  - `published/`：已发布版本与数据复盘
- 命名规范：
  - 草稿：`YYYY-MM-DD-主题短名.md`
  - 配图：`YYYY-MM-DD-主题短名-01.png`

## 8. 指标与复盘
- 核心指标（基础版）：
  - 阅读量、完读率、收藏、分享、评论数
  - 收藏率/在看率（衡量“工具价值感”）
  - 关键词带来的搜索入口占比
- 复盘节奏：
  - 每周小复盘：表现最好/最差各 1 篇
  - 每月大复盘：保留栏目、下线栏目、新增栏目

## 9. 已确认与待确认
1. 仓库类型优先级：`AI 应用 > 内容创作 > 效率工具 > 办公协作`
2. 内容深度：`场景价值导向，不强制实操教程`
3. 执行状态：已创建目录结构与 `templates/` 模板文件
4. 账号目标（前 3 个月）优先级：`涨粉 > 口碑互动 > 私域转化`
5. 选题原则优先级：`有趣与情绪价值 > 场景实用价值 > Trending/Stars`
